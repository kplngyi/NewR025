# 2.4 数据集构建

## 2.4.1 数据来源与组织形式

本研究所使用的数据采用按模态分别存储的组织方式。其中，EEG 数据存放于 `PPEEG/` 目录，fNIRS 数据存放于 `PPfNIRS/` 目录，训练阶段均以 `.fif` 格式文件作为直接输入。文件命名采用“被试编号 + 任务编号 + 模态标识”的方式，例如 `N004_A1L_eeg_raw.fif` 与 `N004_A1R_fnirs_raw.fif`。这种命名方式能够同时保留被试身份、实验条件和信号模态信息，便于后续批量读取、模态配对与结果追踪。

结合 `preprocess.py`、`eeg.py`、`fnirs.py` 以及相关实验 notebook 的调用方式可以确认，当前版本代码稳定支持的数据主要集中于 Action1 条件，即 `A1L` 与 `A1R` 两类实验文件。其中，`A1L` 和 `A1R` 由原始文件名中的 `LeftAction1` 与 `RightAction1` 自动映射得到。因此，从代码实现与可复现性角度出发，当前研究中能够被严格定义的数据基础并不是两个已经独立完成的数据集，而是围绕 Action1 条件构建的肘关节运动数据。

进一步核查仓库中的 EEG、fNIRS 及融合实验文件后可知，当前稳定进入联合分析流程的被试主要包括 `N004`、`N007`、`N008`、`N010`、`N011` 和 `N013`，且均具有 `A1L/A1R` 条件下的有效文件。个别 notebook 中虽然曾出现其他被试编号，但由于未在 EEG 与 fNIRS 两个模态中同时稳定出现，故本文不将其作为正式联合样本统计对象。

## 2.4.2 数据集定义

### 2.4.2.1 数据集 I：肘关节运动数据集

依据现有代码实现，本研究正式构建的数据集应定义为肘关节运动数据集。该定义并非仅来自原始文件标签，而是由训练脚本中的事件重标注与样本构造策略共同决定。在 `eeg.py` 与 `fnirs.py` 中，模型最终使用的类别标签均被统一定义为 `Rest`、`Elbow_Flexion` 和 `Elbow_Extension`，其对应编码分别为 0、1 和 2。由此可见，本研究实际解决的问题是肘关节运动过程中的三分类识别问题，而不是简单的左、右侧动作条件判别。

需要指出的是，文件级标签与样本级标签并不相同。文件级标签反映的是实验原始条件，例如 `A1L` 和 `A1R`；样本级标签则是在读取原始触发信息后，围绕目标事件重新构造得到的三段式时序标签。因此，`A1L/A1R` 主要用于标识实验文件来源，而真正参与模型训练与测试的是 `Rest`、`Elbow_Flexion` 和 `Elbow_Extension` 三类窗口样本。

### 2.4.2.2 关于“数据集 II：手部精细动作数据集”的说明

从论文撰写的严谨性出发，当前版本不宜将“手部精细动作数据集”作为已经完成构建的数据集单独列出。原因在于：其一，`preprocess.py` 仅实现了 `LeftAction1` 与 `RightAction1` 的解析逻辑；其二，代码库中未发现能够稳定支持 `A2`、`Action2`、手指动作或精细手部动作的统一文件命名规则、标签映射规则或训练入口；其三，现有 EEG 与 fNIRS 训练脚本的类别空间均明确指向肘关节屈伸三分类，而非手部精细动作识别。

因此，为保证论文内容与代码实现保持一致，本文仅将“肘关节运动数据集”作为正式定义的数据集，至于手部精细动作数据，在当前版本中尚缺乏充分的代码证据和实验支撑，不纳入本章的数据集定义范围。

## 2.4.3 标签定义与任务编码

### 2.4.3.1 EEG 标签定义

在 EEG 训练脚本中，首先依据原始注释信息提取目标事件。`eeg.py` 中将 `Stimulus/S  5` 作为目标触发描述，并将每次检测到的该类触发时刻视为肘屈曲动作的起始锚点。在此基础上，脚本进一步按照固定时间长度对该锚点进行前后扩展，构造出连续的三段式标签序列：动作前 20 s 定义为静息段 `Rest`，动作开始后 10 s 定义为肘屈曲段 `Elbow_Flexion`，其后连续 5 s 定义为肘伸展段 `Elbow_Extension`。

这一处理方式表明，EEG 数据中的三分类标签并非完全由原始采集设备直接输出，而是由目标事件锚点经过规则化重构后获得。其中，`Rest` 表示围绕动作事件回溯得到的前置静息状态，`Elbow_Extension` 则表示在屈曲阶段结束后顺延生成的后续动作阶段。通过这种方式，可以将原始事件流统一整理为结构明确、类别清晰的三分类任务。

### 2.4.3.2 fNIRS 标签定义

在 fNIRS 训练脚本中，样本标签的构造思想与 EEG 保持一致。`fnirs.py` 中将原始注释字符串 `11` 作为目标锚点事件，并围绕该事件构造 `Rest`、`Elbow_Flexion` 和 `Elbow_Extension` 三类注释段。因此，尽管 EEG 与 fNIRS 在原始注释文本层面并不完全一致，但在进入训练阶段之前，两种模态已经被统一映射到相同的标签空间中。

这种统一标签定义对于后续单模态性能比较以及多模态融合建模具有重要意义。一方面，它保证了 EEG 与 fNIRS 使用相同的行为学语义边界；另一方面，也为后续跨模态特征对齐和决策融合提供了前提条件。

## 2.4.4 EEG 信号预处理

### 2.4.4.1 预处理流程

根据 `preprocess.py`，本研究对 EEG 信号采用了较为规范的基础预处理流程。具体步骤如下：

（1）读取原始 BrainVision 格式 `.vhdr` 文件，并以可写方式加载原始 EEG 数据；

（2）利用自定义电极位置信息文件 `Standard-10-20-Cap81.locs` 完成电极空间位置信息的载入与 montage 设置；

（3）删除不参与脑电建模的辅助通道，包括 `eog1`、`eog2`、`eog3`、`eog4` 以及 `EMG`，以减少眼电和肌电辅助记录对后续模型输入维度的干扰；

（4）对 EEG 信号执行 0.1-40 Hz 带通滤波，以保留与运动任务识别相关的主要低频与中频信息，同时抑制极低频漂移和高频噪声；

（5）进行参考重建。若未显式指定参考通道，则采用平均参考方式对各导联进行重新参考；

（6）采用独立成分分析方法对预处理后的 EEG 数据进行分解，并将 ICA 变换应用于原始信号；

（7）将数据统一重采样至 200 Hz，以降低数据冗余并减轻后续训练的计算负担；

（8）将预处理结果保存为 `.fif` 文件，用于后续事件重标注、滑窗构样与分类训练。

### 2.4.4.2 预处理步骤的作用分析

在脑电信号分析中，预处理的目的在于尽可能保留与认知或运动任务相关的神经信息，同时抑制非神经源噪声、设备漂移以及冗余维度对模型训练的影响。结合本研究的代码实现，上述预处理流程具有以下作用。

首先，利用标准化或半标准化的电极位置信息完成 montage 设置，可以保证不同导联具有明确的空间拓扑意义，为后续空间通道分析、通道筛选以及模型可解释性分析提供基础。对于基于通道选择的 EEG 建模方法而言，电极拓扑信息的合理载入是重要前提。

其次，删除 EOG 和 EMG 等非目标通道能够使模型输入更加聚焦于 EEG 本身。虽然眼电和肌电在伪迹识别中具有一定辅助价值，但若直接保留到分类模型输入中，可能会引入与任务本质无关的噪声来源，甚至使模型过度依赖非脑源特征。因此，在以脑电通道为主要研究对象的任务识别场景中，将其剔除是合理的处理方式。

再次，0.1-40 Hz 带通滤波能够有效抑制基线漂移、缓慢趋势项以及高频干扰。考虑到肘关节运动相关 EEG 信息主要集中于常见节律频段及其附近，当前滤波范围能够较好地覆盖后续特征提取与深度模型训练所需的信息范围。

在重参考方面，平均参考是多通道 EEG 处理中较为常见的策略。其基本思想是以所有导联电位的平均值作为新的参考基准，从而降低单个参考电极异常或局部偏置带来的影响。对于后续涉及通道间差异比较和全头皮信息利用的分析任务，平均参考具有较好的普适性。

独立成分分析的引入，体现了本研究对 EEG 伪迹问题的基本考虑。ICA 的核心思想是将观测到的混合信号分解为若干相互独立的成分，从而为眼动、眨眼、肌电或其他非脑源噪声成分的识别与抑制提供条件。尽管本研究代码中尚未给出完全自动化的成分判别规则，但从流程设置来看，研究已经为后续伪迹处理预留了技术路径。

最后，统一重采样至 200 Hz 可以在尽量保持任务相关时序信息的前提下，减少数据规模和模型训练成本。这一步对于窗口化样本构建和批量神经网络训练尤其重要，因为过高采样率会显著增加样本张量大小，不利于提高实验效率。

### 2.4.4.3 预处理流程的合理性评价

总体而言，当前 EEG 预处理流程满足“从原始记录到可训练数据”的基本要求，能够为后续任务识别提供结构统一、格式规范的输入数据。其合理性主要体现在以下几个方面。

第一，流程完整覆盖了 EEG 预处理中的核心环节，包括电极位置设置、无关通道剔除、频带滤波、重参考、ICA 与重采样，已经具备较强的工程可执行性和实验可复现性。

第二，流程输出文件以 `.fif` 格式统一保存，并在文件名中保留了被试编号和任务标签，便于后续脚本自动检索与批处理调用，这对于构建大规模、可重复的训练流水线具有重要意义。

第三，该流程与后续训练代码之间衔接紧密。`eeg.py` 直接基于 `.fif` 数据读取、重建事件注释并完成窗口化样本生成，说明预处理输出与建模输入之间不存在额外人工转换步骤，减少了人为干预带来的不确定性。

### 2.4.4.4 需要在论文中如实说明的限制

尽管上述 EEG 预处理流程具有较好的基础可用性，但为了保证硕士学位论文表述的准确性，仍需对其局限性作出客观说明。

首先，当前脚本中未实现工频陷波处理。若原始采集环境存在明显的 50 Hz 或 60 Hz 电源干扰，则仍有可能影响部分频段信息的稳定性。

其次，代码中未包含坏导联自动检测与插值策略。对于存在局部接触不良或明显异常波形的导联，当前实现尚未给出系统性处理方法。

再次，虽然脚本调用了 ICA 的 `fit` 与 `apply` 流程，但未给出自动判定伪迹成分的明确规则，`ica.exclude` 仍处于空白占位状态。这意味着 ICA 在当前版本中的应用更接近“流程预留”而非“完全自动化伪迹剔除”。因此，论文中不宜将其表述为已经实现了高质量、全自动的眼电和肌电伪迹清除。

此外，脚本中包含 `raw.plot_sensors(show_names=True)` 的交互式可视化语句，该步骤更适合用于人工检查传感器布局，而非训练流程必需环节。在无图形界面的批处理环境下，该步骤的稳定性和必要性均有限，故在论文中应将其视为辅助检查手段，而不是预处理核心步骤。

综上，本文对 EEG 预处理的准确表述应为：研究完成了面向建模任务的基础 EEG 预处理，并生成可直接用于事件重标注与分类训练的 `.fif` 数据文件；但在坏导联修复、工频抑制和 ICA 自动伪迹识别等方面，当前版本仍存在进一步完善空间。

## 2.4.5 fNIRS 数据使用方式说明

与 EEG 不同，当前仓库中并未提供一个与 `preprocess.py` 对应、且能够完整复现原始 fNIRS 信号处理流程的独立脚本。现有代码能够明确确认的事实主要包括：其一，`fnirs.py` 在训练阶段直接读取 `PPfNIRS/` 目录下的 `.fif` 文件；其二，训练过程中按 HbO/HbR 成对通道进行 Fisher 评分与通道筛选，这说明输入数据中已经包含可区分的 HbO 与 HbR 通道。

然而，从当前仓库内容来看，尚无法确认完整的原始 fNIRS 预处理链条已经在公开代码中实现，例如从原始光强或 `.snirf` 文件读取、光密度转换、基于 Beer-Lambert 定律求解 HbO/HbR、运动伪迹校正、基线漂移校正以及输出标准化 `.fif` 文件等环节，均未在当前版本中形成清晰完整的独立脚本。

因此，本文对 fNIRS 的表述应保持克制和客观，即：本研究在训练阶段直接使用已经整理好的 fNIRS `.fif` 数据，并在建模时按照 HbO/HbR 通道对进行筛选与特征利用；而更早阶段的原始 fNIRS 预处理过程，不宜在论文中写成已经由当前代码完整实现。

## 2.4.6 数据分段与样本构建

### 2.4.6.1 分段原则

在 EEG 与 fNIRS 两个训练脚本中，样本构建方法保持一致。首先，脚本基于目标事件锚点构造 `Rest`、`Elbow_Flexion` 和 `Elbow_Extension` 三类注释段；随后调用 `create_from_mne_raw()` 将每一类注释段进一步划分为多个滑动时间窗样本。窗口长度与窗口步长由配置文件中的 `window_size_samples` 与 `window_stride_samples` 控制，且设置 `drop_last_window=False`，即尽量保留尾部剩余信息。

这种构样方式意味着模型输入的基本单位并不是完整试次，而是从连续信号中截取出的局部时间窗。相较于整段 trial 输入，滑动窗口方式能够在有限数据规模下有效增加样本数量，并增强模型对局部动态模式的学习能力。

### 2.4.6.2 标签编码方式

在样本层面，两种模态采用统一的数值标签编码：`Rest` 对应 0，`Elbow_Flexion` 对应 1，`Elbow_Extension` 对应 2。该编码方式在 EEG 与 fNIRS 中保持严格一致，为后续单模态性能比较、结果统计及跨模态融合分析提供了统一基础。

### 2.4.6.3 数据集划分方式

在 `eeg.py` 与 `fnirs.py` 中，样本划分均基于滑窗后的窗口样本进行。具体而言，脚本先将全部样本按约 70% 与 30% 划分为训练集和临时集，再将临时集进一步等分为验证集和测试集，最终形成约 70%/15%/15% 的训练、验证与测试比例。同时，划分过程中采用 `stratify=labels`，以保持不同类别在各子集中的分布相对一致。

这一方案在类别均衡性和实验实现便捷性方面具有明显优势，适合用于当前阶段的模型训练与参数调节。然而，应注意其本质属于窗口级随机划分，而非 trial 级或被试级独立划分。当相邻时间窗之间存在较高重叠时，训练集与测试集之间可能仍具有一定时间相关性。基于论文规范性要求，本文应明确说明这是一种面向建模验证的默认划分策略，而不是最严格的数据隔离方案。

## 2.4.7 本章小结

综上所述，基于当前代码实现与可复现证据，本文正式定义的数据集应为肘关节运动数据集，而不应将手部精细动作数据作为已完成构建的数据集并列呈现。文件层面采用 `A1L/A1R` 标识实验条件，样本层面则统一转化为 `Rest`、`Elbow_Flexion` 和 `Elbow_Extension` 三分类窗口标签。EEG 预处理流程已经实现基础可用的建模输入生成，尤其在电极定位、滤波、重参考、ICA 和重采样方面形成了较清晰的流水线；但对 ICA 自动伪迹剔除、坏导联处理和工频抑制等问题仍需在后续工作中进一步完善。fNIRS 部分则主要体现为训练阶段对已整理数据的使用，而非完整原始预处理链条的公开实现。上述处理共同构成了本文后续单模态与多模态实验的数据基础。
